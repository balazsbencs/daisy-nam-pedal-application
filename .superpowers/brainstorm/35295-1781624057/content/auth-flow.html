Authenticated

Username shown top-right. Clicking it shows a popover with "Sign out". Tokens stored in Tauri's persistent store; refresh token used to renew on next launch silently.

Token lifecycle

On app launch: Rust checks stored refresh token → silently exchanges for new access token → Discover loads already-logged-in.
On access token expiry: Rust auto-refreshes using refresh token before the next API call (no user action needed).
On refresh failure (revoked / expired): returns to unauthenticated state, prompts re-login.
Sign out: Clears both tokens from the store.
